## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
We need for an agent to be able to ask questions, and we need the system to notify the end user that they need to answer the question.

The question should be asked/answered in the comments thread.

If the agent is waiting for a question to be answered, it should mark the issue with a tag (asking_question), move the task from 'in progress' to 'open', and it should stop the agent. Issues with the 'asking_question' tag must not be moved to 'in progress'.

Once the user answers the question by responding in the chat, the system should remove the tag so the system can pick it back up when an agent is available.

There must be a notification on the dashboard that there are questions to answer, perhaps a drop-down, and selecting one of the issues asking a question causes the issue to appear in the details pane
<!-- SECTION:DESCRIPTION:END -->
